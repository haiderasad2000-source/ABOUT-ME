### ⚙️ Operational Workflows

| Worker | Function | Governance Purpose |
|--------|----------|-------------------|
| `gen-worker` | Automated generation of testing passages | Ensures continuous content pipeline |
| `self-heal` | Continuous validation of ecosystem outputs | Enforces quality and bias compliance meta-rules |
| `retention` | User lifecycle and spaced-repetition | Governs engagement workflows |
| `ft-export` | Anonymized JSONL exports for LLM fine-tuning | Enforces data privacy compliance |
| `mc-validator` | Hourly audit of courses for ambiguity/bias | Continuous educational content compliance |

**Resilience:** If Redis is unavailable, workers gracefully degrade to inline execution—ensuring the platform remains resilient in messy, real-world environments.

### 🛡️ Compliance & Human-in-the-Loop

- **Data Privacy (GDPR/PDPO):** Strict consent gates and native "Right to be Forgotten" endpoints
- **Human Review Queue:** Admin workflow to intercept, manually grade, and override AI outputs
- **Idempotent Webhooks:** Deduplication to prevent double-billing or state corruption
